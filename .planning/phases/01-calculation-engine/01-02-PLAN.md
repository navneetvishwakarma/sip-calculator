**packages/core/tsconfig.json** (D-06 zero-DOM constraint — `"lib": ["ES2022"]`, NO `"dom"`):
```json
{
  "compilerOptions": {
    "target": "ES2022",
    "module": "NodeNext",
    "moduleResolution": "NodeNext",
    "lib": ["ES2022"],
    "strict": true,
    "declaration": true,
    "declarationMap": true,
    "outDir": "dist",
    "rootDir": "src"
  },
  "include": ["src"],
  "exclude": ["node_modules", "dist"]
}
```
Critical: `"lib": ["ES2022"]` with NO `"dom"` is the compile-time zero-DOM gate (per D-06). Any `window`, `document`, or `HTMLElement` reference in `src/` will be a type error. Do not add `"dom"` or `"dom.iterable"` to this array under any circumstance.

Note: `"module": "NodeNext"` and `"moduleResolution": "NodeNext"` are used for direct workspace TypeScript resolution. tsup handles the actual ESM/CJS dual output — it does not use this tsconfig directly.
